Getting Started

When you unzip the download files, you will see two main folders:

  • documentation
  • website files

Obviously, you have now found your way to the documentation. The website files to install on your server are placed inside the website files folder.

Installing the 'PAAY'

To install the 'PAAY' code, place all the files contained in the website files on your web server.

'Neat' is a one-page website, with external page templates for the blog archive and 6 blog post formats.

Sass & CSS files

For ease of customization and flexibility, 'Neat' styling was developped with Sass, which are included in the download files. If you don't know Sass, don't worry - you can leave these files alone and work with CSS. They are here if you want to take advantage of them!

'Neat' is built on the Bourbon Neat framework. You can read more about it on their official website.

The main Sass file that compiles all of the partial files is called main.scss and is located in the /sass/ folder. This Sass file ultimately compiles into the main.css file that is called in the index.html page.

3 extra versions of the main Sass (main-blue.scss, main-red.scss, main-dark.scss) file have been compiled for you. If you want to use one of them instead, all you need to do is call the corresponding css file in index.html.

It is very easy to create your own color style - simply edit the main.scss file and change the $highlight color to your desired HEX value.

Your CSS tweaks shoud be added at the end of the main.scss (or main.css) file, to ensure they override the other styles by being declared after them.

Full-Screen Background Intro

'Neat' starts with a full-screen hero section, where the background is composed of a background image and a background hue color overlay.

The color overlay helps you achieve a high degree of subtility with the background image, allowing to soften heavy colors and fine-tune text readability. To modify it, you can edit the .color-overlay class in the _globals.scss file, or in the main.css file in the 'Template Global Styles' section. You can change the color via background-color and the transparency via opacity - 0 is completely transparent, 1 is opaque.

If you use Sass, you can change the background images in the main.sass file via the parallax variablaes.

Via CSS, the background images can be set in the url() of the background declaration in the sass/partials/_intro.scss file, or in the main.css file under the section.intro .image-bg class.

section.intro .image-bg {
  background: url('../img/name-of-your-image.jpg') 0 0 no-repeat;
}

The same procedure is followed for the other parallax sections - the background image is set in the respective .scss file as shown above.

Finally, the parallax speed can be chaged in the index.html file, in the data-stellar-background-ratio attribute of the .image-bg class.

Make sure that the image you are calling is placed in the correct folder in your server, and since it's a full-screen background, the resolution should be at least 1600px wide by 1200px high or more.

A Sample Plugin

To have the contact form working, you'll need to host your website on an APACHE server that can handle PHP files correctly. It will NOT work locally or on a server that doesn't support PHP.

The main thing you need to do to get the form to work is to enter your desired e-mail address in the php/config.php. This is where the form submissions will be e-mailed. The config.php file contains one line only, don't worry. Replace "your-email-address@here.com" with the appropriate address, that's all you need to do!

If you want to customise the error / success / notification messages, you can do so in /js/script.js (for the success message) and in /php/contact.php and /php/newsletter.php (for the error messages & e-mail confirmation).

The newsletter doesn't automatically sign your users up to a mailing list - it just e-mails you their e-mail address. If you want, you can integrate the form with MailChimp for a more performing setup. Read this tutorial to proceed.

Remember: this form will not work if server doesn't handle PHP files.
